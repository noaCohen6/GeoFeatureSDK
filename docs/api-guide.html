Spring Boot REST API for Feature Management

üéØ Overview

The GeoFeature Backend API is a Spring Boot 3.2 application that provides RESTful endpoints for managing geographic feature toggles. It uses MongoDB Atlas for data storage and is deployed on Koyeb.

üåê Live API: https://thundering-elfie-geofeature-a5030253.koyeb.app
üìö Swagger UI: View API Documentation

üèóÔ∏è Architecture

The API follows a layered architecture:

  • Controller Layer: REST endpoints (FeatureController)
  • Service Layer: Business logic (FeatureService, FeatureServiceImpl)
  • Repository Layer: Data access (FeatureRepository)
  • Model Layer: Domain entities (Feature, GeoRule)
  • Boundary Layer: API DTOs (FeatureBoundary, FeatureQueryRequest)

üìå API Endpoints

Base URL: /api/v1

Method Endpoint Description Auth
GET /features Get all features None
POST /features Create new feature None
GET /features/{id} Get feature by ID None
GET /features/by-name/{name} Get feature by name None
POST /features/query Query feature by country None
PUT /features/{id} Update feature None
DELETE /features/{id} Delete feature None
‚ö†Ô∏è Note: Current implementation has no authentication. For production use, implement proper authentication and authorization mechanisms.

üóÑÔ∏è Data Model

Feature Schema

{
  "id": "string (UUID)",
  "featureName": "string (unique)",
  "defaultStatus": "boolean",
  "geoRules": [
    {
      "countryCode": "string (2-letter ISO)",
      "status": "string (enabled/disabled)",
      "value": "string (optional)"
    }
  ],
  "createdAt": "ISO 8601 datetime",
  "updatedAt": "ISO 8601 datetime"
}

Field Descriptions

  • id: Auto-generated UUID
  • featureName: Unique identifier (e.g., "dark_mode", "payment_methods")
  • defaultStatus: Default on/off for countries without specific rules
  • geoRules: Array of country-specific configurations
  • countryCode: 2-letter ISO country code (IL, US, GB, etc.)
  • status: "enabled" or "disabled" for that country
  • value: Optional custom value (e.g., comma-separated payment methods)

‚öôÔ∏è Setup Instructions

Prerequisites

  • Java 21 or higher
  • MongoDB Atlas account
  • Gradle 8.14 or higher

Local Development

1. Clone the repository:

git clone https://github.com/noaCohen6/GeoFeatureSDK-API.git
cd GeoFeatureSDK-API

2. Configure MongoDB:

Edit src/main/resources/application.properties:

spring.mongodb.uri=mongodb+srv://username:password@cluster.mongodb.net/geofeature_db
server.port=8080

3. Run the application:

./gradlew bootRun

4. Test the API:

curl http://localhost:8080/api/v1/features
‚úÖ Success! API is running at http://localhost:8080
Access Swagger UI at: http://localhost:8080/swagger-ui.html

Docker Deployment

docker build -t geofeature-api .
docker run -p 8080:8080 \
  -e MONGODB_URI="mongodb+srv://..." \
  geofeature-api

üêõ Troubleshooting

Common Issues

Problem: MongoDB connection failed

  • Check MongoDB URI format
  • Verify network access (IP whitelist: 0.0.0.0/0 for testing)
  • Confirm credentials are correct

Problem: API not responding (Koyeb)

  • API is sleeping (scale-to-zero). Open URL in browser to wake (15 seconds)
  • Check Koyeb logs for errors

Problem: CORS errors

  • Verify CorsConfig.java allows your domain
  • Check browser console for specific CORS error
